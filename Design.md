# FlowKit デザインガイド

FlowKit の UI を一貫して保守・拡張するためのデザインシステム定義書です。
本書は「現状のコンセプトとカラーを尊重したうえで、デザイナー観点での監修・是正を加える」ことを目的にまとめています。

- 対象範囲: `src/app/globals.css`（デザイントークン本体）、`src/lib/theme.ts`（アクセント定義）、`src/components/**`
- 配色モデル: すべて **OKLCH**（知覚均等色空間。明度 L をそろえれば見た目の明るさがそろう）
- 実装基盤: Next.js (App Router) + Tailwind CSS v4 (`@theme inline`) + shadcn/ui ベースのプリミティブ

---

## 1. デザインコンセプト

> **「業務ツールを、説明書なしで迷わず使える静かなプロダクトに」**

FlowKit は Power Automate の非自明なパッケージ生成を、ステップ式ウィザードで誰でも扱えるようにするツールです。
デザインはその目的を支える次の 4 原則に従います。

| 原則 | 意味 | UI への落とし込み |
| --- | --- | --- |
| **Calm / 静けさ** | 情報の主役は入力フォーム。装飾は背景に退く | ニュートラル基調 + 単一アクセント。背景はごく淡いグロー + ドットのみ |
| **Guided / 導線** | 「今どこにいて、次に何をするか」が常に明確 | 上部ステッパー、`Step n / N`、固定フッターの主従ボタン |
| **Trust / 信頼** | 業務データを扱う安心感（"AI 不使用・ブラウザ内完結"） | 端正なグリッド、等幅フォントによる ID/時刻表示、過度でない余白 |
| **Responsive Motion / 適応する動き** | 演出は理解を助ける範囲で。邪魔をしない | 方向性のあるトランジション。低性能・縮小設定・OS 設定で自動減衰 |

「コンセプトとカラーは現状維持」が前提のため、本書は**配色アイデンティティ（ブランド/ベース色相）を変更しません**。
監修では主にアクセシビリティ・一貫性・トークン整理・潜在バグの是正を扱います（§11 参照）。

## 2. ブランド

- **ワードマーク**: `Flow` + `Kit`（`Kit` をアクセント色で強調）。`.brand-wordmark` / `.brand-wordmark-accent`
- **シンボル**: 角丸スクエアにグラデーション + `Workflow` アイコン（`.brand-mark`）
- **タグライン**: 「Box / SharePoint の Excel テーブルを CSV 化して定期出力」

監修メモ:
- ワードマークのフォールバックに `Arial Black` が入っているが、第一フォントは Inter 800。OS により字幅差が出るため、本来は Inter（または可変フォント）に寄せるのが望ましい（識別性に影響しない範囲のため今回は据え置き）。
- `letter-spacing: -0.055em` は小サイズで詰まりすぎないか定期的に確認すること。

## 3. カラーシステム

### 3.1 セマンティックトークン（ライトテーマ）

`:root` で定義し、`@theme inline` 経由で Tailwind ユーティリティ（`bg-background` 等）に接続。

| トークン | 値 (OKLCH) | 役割 |
| --- | --- | --- |
| `--background` | `0.99 0.002 270` | ページ地 |
| `--foreground` | `0.21 0.015 270` | 本文 |
| `--card` / `--popover` | `1 0 0` | カード・ポップオーバー地 |
| `--muted` | `0.968 0.004 270` | 控えめな面 |
| `--muted-foreground` | `0.48 0.012 270` | 補助テキスト（**監修で 0.52→0.48 に調整**, §11） |
| `--border` / `--input` | `0.922 0.005 270` | 罫線・入力枠 |
| `--destructive` | `0.58 0.2 25` | エラー・破壊的操作 |
| `--primary` / `--ring` | `var(--brand)` | 主ボタン・フォーカスリング |

設計意図: 色相をニュートラル側もわずかに `270`（青紫）へ寄せ、ブランド（violet 既定）と地続きの "冷たすぎない無彩色" を作っている。色温度の統一は今後も維持すること。

### 3.2 ブランド（アクセント）トークン

ランタイムで差し替え可能な 6 プリセット（`src/lib/theme.ts`）。既定は **violet**。

| トークン | 役割 |
| --- | --- |
| `--brand` | アクセント主色（テキスト/アイコン/枠） |
| `--brand-strong` | グラデーション終点・濃色 |
| `--brand-soft` | 選択中チップ/バッジの淡い面 |
| `--brand-softer` | 入力強調ブロックの極淡面 |
| `--brand-ring` | フォーカス/選択リング（alpha 0.3 前後） |
| `--brand-foreground` | アクセント上の前景（ほぼ白） |
| `--brand-glow` | 背景グロー・影の色味 |

プリセット: `violet / blue / teal / emerald / amber / rose`。
**設計ルール**: 各プリセットは `--brand` の明度をおおむね `L≈0.57–0.66` にそろえ、`--brand-soft` は `L≈0.965` に固定。これにより accent を切り替えても**コントラスト体験が崩れない**。新色追加時もこの L 帯を踏襲すること。

### 3.3 コントラスト基準（WCAG）

| 組み合わせ | 目標 |
| --- | --- |
| 本文 `foreground` / `background` | AAA 相当（十分） |
| 補助 `muted-foreground` / `background` | **AA (4.5:1) 以上**を死守。`L` を下げて担保 |
| アクセント上テキスト（白） | AA 以上。`--brand` の L 帯で担保 |
| プレースホルダ | 視認可能（監修で `0.74→0.62` に調整, §11） |

> アクセント文字をニュートラル地に置く場合（`text-[var(--brand)]`）、特に `amber` は明度が高めなので小サイズの多用は避け、アイコン/強調語に留める。

## 4. タイポグラフィ

### 4.1 フォントスタック

| 用途 | 変数 | フォント |
| --- | --- | --- |
| 欧文サンセリフ | `--font-inter` | Inter |
| 和文（既定/可変） | `--font-jp` | Zen Kaku Gothic New（設定で Noto Sans JP / Zen Maru / メイリオに切替） |
| 等幅 | `--font-mono-src` | JetBrains Mono |

`--font-sans = Inter, 和文, system-ui …` の順。日本語は `font-feature-settings: "palt" 1` で約物アキを詰め、`letter-spacing: 0.01em` で欧文の窮屈さを緩和。

### 4.2 タイプスケール（現行の実値）

| 役割 | サイズ | ウェイト | 備考 |
| --- | --- | --- | --- |
| 画面タイトル (h1) | `1.5–1.75rem` | 600 | `tracking-tight` |
| カード見出し (h2) | `1.125rem` | 600 | |
| 本文 | `0.875rem` | 400/500 | UI 標準 |
| 補助 | `0.75rem` | 400/500 | キャプション・注記 |
| Eyebrow/ラベル | `0.6875rem` | 600 | `uppercase` + `tracking 0.12–0.14em` |
| 数値ディスプレイ | `2.25–3rem` | 600 | 等幅 + `tabular-nums` |

**`.mono`**: ID・時刻・ファイル名など「機械的な値」は必ず等幅 + `tabular-nums` で表示し、桁ブレを防ぐ（信頼感の演出）。

監修メモ: サイズが各所でハードコードされている。将来的には `--text-xs … --text-2xl` のトークン化を推奨（§11 ロードマップ）。

## 5. スペーシングとレイアウト

- 基本リズムは Tailwind の 4px グリッド（`gap-2 / p-6 / space-y-6` …）。
- **コンテンツ幅**: メイン `max-w-2xl`（フォームの可読幅）、ヘッダー `max-w-5xl`。
  - 監修メモ: 幅が異なるため広い画面でブランドとカードの左端が不一致。意図的な"ヘッダーは広く・本文は集中"の構成だが、`max-w-3xl` への寄せ or ヘッダー内をカード幅にそろえる選択肢も検討余地あり（今回は据え置き）。
- **カード内パディング**: `p-6`（モバイル）→ `sm:p-8`。
- セクション区切りは `border-t border-border` + 上下マージンで"段"を作る（`.prose-section`）。

## 6. 角丸・エレベーション・サーフェス

### 6.1 角丸スケール

`--radius: 0.75rem` を基準に派生（`sm = r-4 / md = r-2 / lg = r / xl = r+4 / 2xl = r+12`）。
ピル形状（ステッパーのバブル、チップ、スイッチ）は `999px`。

### 6.2 影

| トークン | 用途 |
| --- | --- |
| `--shadow-soft` | 通常のカード（`.surface`） |
| `--shadow-elevated` | ドロワー/ポップオーバー等の浮遊面 |

影は無彩色ではなく `foreground` 色相のアルファで落とし、地の色温度となじませている。

### 6.3 サーフェス（`Surface`）

| variant | クラス | 見た目 |
| --- | --- | --- |
| `glass`（既定） | `.surface` | 半透明 + `backdrop-filter: blur(16px)`。背景グローを透かす主役カード |
| `solid` | `.surface-solid` | 不透明カード。ガラスが不要な静的面（**監修で命名不整合を修正**, §11） |

> ガラス面の上に `backdrop-filter` を持つ別レイヤー（ヘッダー等）を重ねると描画不具合が起きやすい。重ね順に注意（過去に設定ドロワーで対応済み）。

## 7. モーション

| 変数/クラス | 値 | 用途 |
| --- | --- | --- |
| 標準イージング | `cubic-bezier(0.22, 1, 0.36, 1)` | 進入（decelerate） |
| 退出イージング | `cubic-bezier(0.4, 0, 1, 1)` | 退出（accelerate） |
| ビュー切替 | spring `stiffness 280 / damping 32`（通常）, 0.28s tween（compact） | 画面遷移 |
| ステップ切替 | spring `320 / 34`（通常）, 0.18s（compact） | ウィザード内 |

原則:
1. **方向性**: 「次へ」は右→左、「戻る」は左→右。空間的な前後関係を体で覚えさせる。
2. **適応**: `useCompactMotion()` がモバイル/`pointer:coarse` を検知し、blur やオフセットを抑制（GPU 負荷と酔いの低減）。
3. **減衰**: 設定の「効果」OFF（`data-animations="off"`）と OS の `prefers-reduced-motion` の**両方**で無効化。アクセシビリティ必須要件。

## 8. アイコン

- ライブラリ: **lucide-react**。標準サイズ `h-4 w-4`（16px）、補助 `h-3.5`（14px）。
- 線幅は lucide 既定（2）。色は文脈に応じ `text-[var(--brand)]` / `text-muted-foreground`。
- 装飾アイコンには必ず `aria-hidden`。意味を持つボタンには `aria-label`。

## 9. コンポーネント仕様

### 9.1 ボタン（`ui/button.tsx`）

- variant: `default / destructive / outline / secondary / ghost / link`
- size: `xs / sm / default / lg / icon*`
- インタラクション: hover で `-translate-y-px`、active で `scale 0.97`、`focus-visible` でリング（`ring-ring/50`）。
- 主従: 1 画面の主アクションは `default` を 1 つ。副は `ghost`/`outline`。

### 9.2 入力（`ui/input.tsx`）

- 高さ `h-9`、`focus-visible` で枠 + 3px リング。モバイルは `text-base`（iOS の自動ズーム回避）。
- ID 系入力は `.mono` を付与。

### 9.3 セグメンテッドコントロール（`.segment` / `.segment-item`）

- データソース選択・頻度・文字サイズで使用。選択中は `--brand` 文字 + `--card` 面 + ring線。
- **監修で `:focus-visible` アウトラインを追加**（§11）。

### 9.4 ステッパー（`.step-bubble` / `.step-line` / `.step-caption`）

- 状態 `todo / active / done`。active はグラデ + `scale 1.06`、done はチェック。
- 到達済みステップのみクリック可。キャプションは `sm` 以上で表示。
- **監修で到達済みバブルに `:focus-visible` を追加**。

### 9.5 時刻入力（`.time-field` / `.step-btn` / `.time-display`）

- 上下ボタン + ホイール操作。分は 5 分刻み。大きな等幅数値で現在値を表示。
- `title` でホイール操作を示唆。`aria-label` 完備。

### 9.6 チップ（`.preset-chip`）

- よく使う時刻 / 曜日選択。選択中は `--brand-soft`。`:focus-visible` 追加済み。

### 9.7 ナビゲーション

- デスクトップ: ヘッダー水平ナビ（`.nav-link`、アクティブ下線）。
- モバイル: 右スライドのドロワー（`NavDrawer`）。オーバーレイ + `translate-x`。
- 設定: 右スライドシート（`SettingsButton`、`createPortal` で body 直下に描画）。

### 9.8 カード/サマリ/空状態

- 保存済みフロー・確認サマリは `dl`/`grid` で「キー: 値」を整列。値は `.mono`。
- 空状態は中央寄せのアイコン + 見出し + 補助文（`SavedFlowsView`）。

## 10. 状態とインタラクション

| 状態 | ルール |
| --- | --- |
| hover | 色 or 微小な持ち上げ。レイアウトを動かさない |
| active | `scale` ダウンで押下感 |
| **focus-visible** | **全インタラクティブ要素に可視リング必須**（キーボード操作の生命線） |
| disabled | `opacity 0.5` + `pointer-events-none`、ホバー演出も停止 |
| selected | アクセント面 + ring 線で「いま選ばれている」を明示 |
| error | `--destructive` 文字、`aria-invalid` 連動の枠/リング |

## 11. デザイン監修（本レビューでの是正点）

「コンセプト・配色アイデンティティは維持」を守りつつ、プロダクト品質に直結する以下を是正しました。

### 実施済み（このコミット）

1. **キーボードフォーカスの可視化（最重要 / a11y）**
   - 自作コントロール（セグメント、ステッパーバブル、時刻ボタン、チップ、ナビリンク、ヘルプドット、ドロワー項目、設定の各ボタン/スイッチ/カラースウォッチ）に `:focus-visible` を追加。
   - 共有トークン `--focus-width` / `--focus-offset` を新設し、`outline` ベースで統一。
   - 是正理由: 既存はネイティブボタン以外にフォーカス表示がなく、キーボード/支援技術ユーザーが操作位置を見失う状態だった。

2. **潜在バグ修正: `Surface` の variant 不整合**
   - コンポーネントは `surface-solid` を出力する一方、CSS は `.surface-quiet` を定義しており `variant="solid"` が無効化されていた。CSS 側を `.surface-solid` に統一。

3. **コントラスト調整（a11y / 配色アイデンティティは不変）**
   - `--muted-foreground` `L 0.52 → 0.48`（補助テキストを WCAG AA ≈4.5:1 へ）。
   - プレースホルダ `L 0.74 → 0.62`（淡すぎる例示文の視認性確保）。
   - いずれも色相・彩度は据え置きで、ブランド/ベースの色は変更していない。

4. **ターゲットサイズ**: ヘルプドットの当たり判定を `17px → 24px`（負マージンでレイアウト不変）。

5. **コピーの一貫性**: テーブルベース変換へ仕様変更したのに残っていた「シート」表記を「テーブル」に統一（ウィザードのステップ名/確認/説明）。

### 推奨ロードマップ（未実施・要判断）

| 優先 | 項目 | 内容 |
| --- | --- | --- |
| 高 | **ダークモード** | shadcn の `dark:` クラスと `viewport.themeColor` は存在するが、`.dark` の配色トークンが未定義で実質非対応。OKLCH なので L 反転ベースで体系的に追加可能。 |
| 中 | **タイプスケールのトークン化** | サイズのハードコードを `--text-*` に集約し、文字サイズ設定との整合を一元化。 |
| 中 | **スペーシングトークン** | `0.7rem` 等のアドホック値を 4px グリッドへ寄せ、独自値はトークン化。 |
| 低 | **レイアウト幅の整合** | ヘッダー(`5xl`)と本文(`2xl`)の左端不一致の解消可否を検討。 |
| 低 | **ワードマークのフォント** | `Arial Black` フォールバック依存をやめ Inter/可変フォントへ。 |

## 12. 運用ルール（新規実装時のチェックリスト）

- [ ] 新色は §3.2 の **L 帯**（brand `0.57–0.66`、soft `≈0.965`）を守る
- [ ] テキスト/地のコントラストは補助文でも **AA 4.5:1** を満たす
- [ ] 機械的な値（ID/時刻/ファイル名）は `.mono`
- [ ] すべてのインタラクティブ要素に **`:focus-visible`** がある
- [ ] 当たり判定は **24px 以上**（モバイル主要操作は 40px 以上が望ましい）
- [ ] アニメーションは `data-animations` と `prefers-reduced-motion` の両方で減衰する
- [ ] 装飾アイコンに `aria-hidden`、操作要素に `aria-label`
- [ ] 色は生値ではなくセマンティック/ブランドトークン経由で参照する
